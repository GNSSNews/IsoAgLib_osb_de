\subsection CanServerUsage CAN Server Usage
To see all available command line switches of <tt>can_server</tt>/<tt>can_server.exe</tt>, run it with <tt>--help</tt> as parameter:
\code
usage: can_server [--log <log_file_name_base>] [--file-input <log_file_name>] [--high-prio-minimum <num_pending_writes>] 
                  [--reduced-load-iso-bus-no <bus_number>

  --log                      log can traffic into <log_file_name_base>_<bus_id>
  --file-input               read can data from file <log_file_name>
  --high-prio-minimum        if 0: start normally without priority-handling (default - used if param not given!).
                             if >0: only clients with activated high-priority-send-mode can send messages if
                                    can-controller has equal or more than <num_pending_writes> in its queue.
  --reduced-load-iso-bus-no  avoid unnecessary CAN bus load due to
                             messages with local destination addresses
  --nice-can-read            set a nice value for can read thread to reduce slowing down effect on other applications
                             due to heavy OS task switches, 0 < value < 20 for reducing the priority
\endcode
